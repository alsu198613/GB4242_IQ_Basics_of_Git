1.Делаем fork 
2.Создаем папку 
3.Инициализирум ее git init
4.Пишем в терминале git clone ссылка
5.Создаем ветку git checkout -b имя ветки                 - создать ветку с переоходом на нее

git init                                  - инициализация(создание локального репозитория) 
git status                                - показывает текущий статус(подготовленные файлы пред коммитом)
git log                                   - выводит всю инфу о комитах
git log --oneline                         - выводит только названия комитов и их хеш
git log --all --oneline                   - выводит все созданные ветки,название коммитов и их хеш
git show                                  - показывает пследний коммит
git add <file>                            - сохраняет в индексе изменения(пдготавливает к комиту)
git add .                                 - сохраняет в индексе все изменения
git clone ссылка на github                - клонирует из github репозиторий
git commit -m "text"                      - создаем комит(сохраняем)(преносим из индекса в постоянное хранение)
git commit -am "text"                     - то же самое ,только можно не сохранять в индексе заранее 
git push -u origin имя ветки              - 1 раз,отправить изменение в github(устанавливает авто связь между локальным и удаленным репозиторием)
git push origin имя ветки                 - отправить изменение в github 
git pull                                  - втянуть обновленный репозиторий(комит) с удаленного репозитория
git --version                             - версия гита и проверка его есть ли он вообще
git config --global user.name "name"      - настройка имени пользователя 
git config --global user.email "email"    - настройка почты пользователя
git config --list                         - выводит все настройки гита
git remote add origin https://.....       - установить связку с удаленным репозиторием
git remote -v                             - вывести путь к удалённому репозиторию
git remote remove origin                  - удалить путь к удаленному репозиторию  
origin                                    - название удаленного репозитория по умолчанию 
git branch имя ветки                      - создать ветку без перехода на нее
git checkout -b имя ветки                 - создать ветку с переоходом на нее
git checkout имя ветки                    - переход между ветками
git branch                                - вывод всех веток
git branch -a                             - вывод всех веток(зеленые локальные, красные удаленные)
git stash                                 - откладываем внесенные изменения которые небыли закомиченны(прячем)
git stash pop                             - возобновляем спрятанные изменения
git stash list                            - вывод наборов изменений
git stash drop номер                      - удаление stash

.gitignore                                     - игнорируем картинки,логи,большие файлы и т.д
В этот файл добавляем :
имя файла.расширение
*.расширение                                   - * значит игнорим все файлы с расширением
имя папки/                                     - / значит инорим всю выбрпнную папку
Чтобы не игнорировать каки то файлы в паке :
папка/*.расширение
!имя папки/имя файла.расширение                - ! значит не игнорим выбрaнный файл
